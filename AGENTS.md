# CDE (Common Desktop Environment) — Agent Guide

## What This Is

CDE is the open-sourced (LGPL v2) X11 desktop environment originally from HP/IBM/Sun/Novell. The git repo contains the full source: ~2300 C files, ~1700 headers, ~960 C++ files, plus shell scripts, yacc/lex grammars, message catalogs, and localization data. Version 2.5.3 is current on `master`.

## Build System

**Autotools (autoconf + automake + libtool).** No CMake, no Meson.

```
cd cde/
./autogen.sh          # runs libtoolize, aclocal, autoconf, autoheader, automake
./configure            # standard autoconf; see options below
make                   # or: make World (for debian builds)
sudo make install      # installs to /usr/dt by default
```

**BSD:** must use gmake and specify TCL location:
```
./configure --with-tcl=/usr/local/lib/tcl8.6 MAKE="gmake"
gmake
sudo gmake install
```

### Configure Options

- `--enable-german` / `--enable-italian` / `--enable-french` / `--enable-spanish` / `--enable-japanese` / `--enable-chinese` / `--enable-chinese-tw` — build locale data (off by default)
- `--disable-docs` — skip building help/dtinfo docs
- `--disable-xrender` — disable Xrender extension
- `--enable-misc-fixed` — use misc-fixed font
- `--with-pam-dir=DIR` — PAM config directory (default `/etc/pam.d`)
- `--with-tcl=DIR` — TCL library path (BSD)
- `--with-utmp-file=PATH` / `--with-wtmp-file=PATH` — sessreg paths

### Required Build Dependencies

X11 (Xlib, Xt, Xmu, Xext, Xpm, Xinerama, Xau, Xdmcp, Xm/Motif), libjpeg, liblmdb, libiconv, tirpc (or legacy RPC), flex/bison (or byacc), ksh/ksh93/mksh, rpcgen, gencat, onsgmls, bdftopcf, mkfontdir, gzip, m4, sessreg, compress, perl, xrdb, cpp. PAM on Linux/NetBSD.

### Key Build Variables (configure.ac)

| Variable | Default | Purpose |
|---|---|---|
| `CDE_INSTALLATION_TOP` | `/usr/dt` | Install prefix |
| `CDE_CONFIGURATION_TOP` | `/etc/dt` | Config directory |
| `CDE_LOGFILES_TOP` | `/var/dt` | Log directory |
| `CDE_USER_TOP` | `.dt` | Per-user dot directory |

### Build Gotchas

- `tradcpp` (in `cde/util/tradcpp/`) is built first and used as the C preprocessor for generating localized configs — it is NOT gcc -E. The `GENCPP` variable points to the built tradcpp.
- `include/config.h` is a stub that forwards to `cde_config.h` (generated by autoconf). Motif's private headers include `<config.h>`.
- Some generated files are committed and tracked in `.gitignore` (e.g., `lib/DtMmdb/StyleSheet/defParser.C`, yacc output, MsgCat.c copies). These are regenerated by the build.
- `make World` is needed for debian builds, not just `make`.
- Debian packaging uses `admin/IntegTools/dbTools/installCDE` for install (standard `make install` is "broken" per `debian/rules`).

## Architecture Notes

- **Libraries are built as libtool `.la` archives**, then linked into programs. Library dependency chain: `DtSvc → DtHelp → DtPrint → DtWidget → DtTerm → DtMrm → tt (ToolTalk) → DtSearch → csa → DtMmdb → DtXinerama`.
- **ToolTalk (`lib/tt/`)** is the RPC-based inter-application messaging system. Uses Sun RPC / tirpc. Has its own type compiler (`tt_type_comp`), session daemon (`ttsession`), and file-scoping daemon.
- **dtmail and dtcm are C++** programs (`.C` files). Most everything else is C.
- **dtappbuilder** has auto-generated UI code (`*_ui.c`, `*_stubs.C`) from the Motif UIL compiler.
- **dtlogin** requires PAM on Linux/NetBSD; falls back to `/etc/passwd` elsewhere.
- **Localized resources** (`programs/localized/`) use a three-file pattern per locale: `.nls` (source), `.tmsg` (translated messages), compiled `.cat` (message catalog via `gencat`).
- **The `imports/motif/` directory** contains bundled Motif headers and localized data needed for building but not distributed separately.

## Code Conventions

- **License header**: Every `.c`/`.h` file has the LGPL v2 copyright block (Open Group + original vendor copyrights: HP, IBM, Sun, Novell).
- **Naming**: `Dt` prefix for public APIs (`DtMainLoop`, `DtDbLoad`, etc.), `_Dt` for private/internal functions.
- **Header guards**: Traditional `#ifndef _DtFoo_H / #define _DtFoo_H` style.
- **C standard**: Code targets ANSI C with K&R-era legacy. Recent work (2024-2025) has been migrating to C11/C23 strict prototypes. GCC 15 compatibility fixes are in progress.
- **Compiler flags** (set in configure.ac): `-Wno-format-truncation`, `-Wno-write-strings`, `-Wno-unused-result`, `-fno-strict-aliasing`. These are intentional suppressions for legacy code.
- **Platform conditionals**: Extensive use of `#ifdef sun`, `#ifdef __linux__`, `#ifdef CSRG_BASED` for BSD, plus automake conditionals (`LINUX`, `FREEBSD`, `OPENBSD`, `NETBSD`, `SOLARIS`, `AIX`).
- **Yacc/Lex**: Several parsers use `.y`/`.l` files (DtSearch boolean parser, ToolTalk type compiler, dtcm date parser, dtappbuilder BIL parser). These are processed by bison/flex during build.

## Testing

There is **no automated test suite**. Testing is manual:
- `cde/programs/dtterm/tests/` has interactive terminal test scripts (not unit tests).
- `cde/programs/dtksh/ksh93/src/cmd/ksh93/tests/` has ksh93 regression tests (from the upstream ksh project).
- `cde/lib/tt/demo/` has ToolTalk demo apps (standalone Makefiles, not integrated).
- `cde/programs/dtappbuilder/src/` has isolated `*_test.c` files for some libAB modules.
- The `cde/examples/` directory has sample CDE programs.

## Common Tasks

### Adding a new locale
1. Add locale directory under `cde/programs/localized/<locale>/` with subdirs: `app-defaults/`, `backdrops/`, `config/`, `msg/`, `palettes/`, `types/`
2. Add `AC_ARG_ENABLE` in `configure.ac` and `AM_CONDITIONAL`
3. Add `Makefile.am` and locale's `Makefile` to `AC_CONFIG_FILES` in `configure.ac`
4. Add locale to `cde/programs/localized/Makefile.am` SUBDIRS

### Adding a new program
1. Create directory under `cde/programs/<name>/`
2. Write `Makefile.am` following existing patterns (see `dthello/` for simplest example)
3. Add to `cde/programs/Makefile.am` SUBDIRS
4. Add `cde/programs/<name>/Makefile` to `AC_CONFIG_FILES` in `configure.ac`

### Modifying a shared library
- Library Makefile.am files use `lib_LTLIBRARIES` with libtool
- CPPFLAGS include `-I./include -DMULTIBYTE -DNLS16 -I../include @DT_INCDIR@`
- `CDE_INSTALLATION_TOP` and `CDE_CONFIGURATION_TOP` are substituted into source via `AC_SUBST`

## Versioning

Tags: `2.3.0`, `2.3.1`, ..., `2.5.3` (stable), plus `v1.0.x` for something separate. The `master` branch is the active development branch. Version macros are in `configure.ac`: `CDE_VERSION_MAJOR=2`, `CDE_VERSION_MINOR=5`, `CDE_VERSION_MICRO=3`.

## Things Likely to Trip You Up

- **Don't `make clean` and expect everything to rebuild correctly** — some generated files are in gitignore and need the full build chain. Use `make World` for clean rebuilds.
- **The `imports/motif/` directory** is for build-time only — Motif headers that CDE compiles against. Don't confuse with system Motif.
- **MsgCat.c** appears in multiple programs (`dtdspmsg/`, `dtexec/`, `dtsr/`, `localized/util/`) — these are copies of the same file, not independent implementations.
- **dtinfo is the most complex program** — it has its own MMDB library, OLIAS database, and multiple sub-Makefiles. Changes here are fragile.
- **The `tradcpp` utility** replaces imake's C preprocessor role. If you see `GENCPP` in a Makefile, it references `$(top_builddir)/util/tradcpp/tradcpp`.
- **`config.h` at repo root (`include/config.h`)** is just a 6-line stub forwarding to `cde_config.h`. Do not modify it.
- **BSD builds** require `gmake` and explicit `--with-tcl` path. Regular `make` won't work.
- **Locale data is huge** — the `.gitignore` lists thousands of generated locale files. Don't be surprised by the size.
- **The `historical/` directory** contains old release notes for pre-opensource versions (2.1.x) — not relevant to current development.
