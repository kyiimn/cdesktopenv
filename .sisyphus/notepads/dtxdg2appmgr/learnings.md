# Learnings - dtxdg2appmgr Task 1

## Build System Integration
- AC_CONFIG_FILES requires the Makefile.am to exist even for conditional directories — autoreconf fails with "required directory does not exist" if the directory is missing
- Created a minimal placeholder Makefile.am (just MAINTAINERCLEANFILES) so autogen.sh passes; full content will be added in Task 2
- `PKG_CHECK_MODULES` uses `--without-xdg-integration` (AC_ARG_WITH, not AC_ARG_ENABLE) — convention uses `--with/--without` for optional feature dependencies
- When `--without-xdg-integration` is set, PKG_CHECK_MODULES is skipped entirely (no pkg-config call), and `have_glib=no` is set directly
- The `HAVE_GLIB2` automake conditional pattern follows the same style as `HAVE_XFT` on line 616
- GLIB_CFLAGS and GLIB_LIBS are AC_SUBST'd so they'll be available in Makefile.am files via @GLIB_CFLAGS@ and @GLIB_LIBS@
## Task 2: Directory structure and stubs

- CDE convention is `#include <cde_config.h>` (not `<config.h>`) when using `HAVE_CONFIG_H` guard. The dsdm/dsdm.c program follows this pattern.
- `cde_config.h` is generated by autoconf from `cde_config.h.in` and lives at `cde/include/cde_config.h`.
- The `include/config.h` at repo root is a 6-line stub forwarding to `cde_config.h`.
- When HAVE_GLIB2 is false, `bin_PROGRAMS` gets commented out in generated Makefile — the program is silently skipped.
- GLib 2.0 was available on this system via pkg-config but HAVE_GLIB2 conditional wasn't set because configure needs re-run after Task 1 changes.
- Manual compilation with `gcc -I$(cde/include path)` works for verifying source correctness.
- The `dtxdg2appmgr_` prefix for public functions avoids conflicts with `_Dt` private namespace.

## Task 3: CLI Options Parser (options.c / options.h)

- GOption API pattern: use `static GOptionEntry[]` with NULL sentinel, then set `.arg_data` to local variables in the parse function (not to struct fields directly). This avoids the struct layout dependency issue and lets GOption allocate strings.
- GOption allocates strings for `G_OPTION_ARG_STRING` entries via glib — the caller must NOT free them if they end up stored in the struct (GOption owns them). But since we `g_strdup()` defaults when arg is NULL, the caller must `g_free()` all string fields in `_options_free()`. For user-provided strings, GOption allocates them and we store the pointer directly — `g_free()` is still correct since GOption uses `g_malloc()`.
- `g_option_context_parse()` returns FALSE on error (unknown option, missing arg) and sets `GError`. The error message includes the unknown option name. Print it with `g_printerr()`.
- `#ifndef CDE_INSTALLATION_TOP` / `#define` guards in options.c provide fallback defaults when the macro isn't set by the build system — needed for standalone test compilation.
- C string literal concatenation (`CDE_INSTALLATION_TOP "/appconfig/appmanager"`) works because the preprocessor substitutes the macro as a string literal, and adjacent C string literals are concatenated.
- `dtxdg2appmgr_parse_options` returns NULL on parse failure, and main.c already checks `if (opts == NULL)` and calls `dtxdg2appmgr_print_usage(argv[0])` then returns 1. This means invalid options get both the GOption error message AND our usage text.
- The `retain` field (gboolean for `-r`) mirrors dtappgather's `-r` flag which retains previous Application Manager contents.
- LSP (clangd) shows errors for GLib types (`gboolean`, `gchar`, `GOptionEntry`) because it doesn't have the glib include paths configured. The actual build with Makefile CPPFLAGS works correctly.
