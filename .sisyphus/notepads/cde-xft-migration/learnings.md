## instant program UTF-8 mblen bug fix (2026-06-14)

### Problem
`cde/programs/dtdocbook/instant/util.c` `Putc()` function returns EOF when `mblen(argBuf, i) == -1` and `i >= MB_CUR_MAX`. This causes `instant` to exit with code 1, which makes `dtdocbook2sdl` fail with "Error processing ... by instant" when building fr_FR.UTF-8 help files (CreatAct, Intromgr, etc.) that contain UTF-8 characters not representable in the current shift state.

### Root Cause
The SGML parser (`dthelp_htag2`) feeds bytes one at a time. When the parser is in a state where `LC_ALL=C.utf8` is active and `MB_CUR_MAX=6`, the parser may emit bytes that are not valid in the current sequence. The original code returned EOF which propagated up as a fatal error.

### Standard CDE Pattern
The established CDE pattern (used in `cde/programs/dthelp/parser/*/util/scanutil.c:30-72`) handles this case by:
1. Keeping the warning message ("An invalid multi-byte character was found in the input")
2. Setting a fallback character (e.g., c=' ' and length=1)
3. Continuing processing instead of returning EOF

### Fix Applied
In `cde/programs/dtdocbook/instant/util.c` `Putc()`:
- Removed `i = 0;` and `return EOF;` from the `else` branch
- Let execution fall through to the existing `Tcl_ExternalToUtfDString` path
- The for-loop `for (j = 0; j < i; ++j) *pc++ = argBuf[j];` writes all accumulated bytes as single characters
- The `i = 0;` reset at line 738 (just before closing the Tcl command) handles the reset

### Verification
- `LC_ALL=C.utf8 make CreatAct.sdl` → exit 0, 69625 bytes
- `LC_ALL=C.utf8 make Intromgr.sdl` → exit 0, 169826 bytes
- `make Appmanager.sdl` (en_US) → exit 0, 79464 bytes
- Full `LC_ALL=C.utf8 make` in fr_FR.UTF-8/help → 21 .sdl files, no errors
- Full `make` in en_US.UTF-8/help → 19 .sdl files, no errors
- Full `make` in de_DE.UTF-8/help → no errors

### Key Technical Insight
`MB_CUR_MAX=6` for UTF-8 locales in glibc, and `argBuf[8]` is sized safely. The static `commandBuf` is `"OutputString \"                 "` (with trailing spaces) which is then null-terminated, so writing i=6 bytes plus a quote is safe.

## SDL help build locale fix (2026-06-14)

### Problem
`make dist` failed for `de_DE.UTF-8/help` because `dthelp_htag2` binary calls `setlocale(LC_ALL, "de_DE.UTF-8")` which silently fails when the locale doesn't exist on the system. The `LANG=de_DE.UTF-8` variable was exported by the help Makefile.

### Fix Applied
Added `LC_ALL=C.utf8` prefix to the `%.sdl` suffix rule in `cde/doc/common/help/sdl-docs.am`:
```makefile
%.sdl:
	LC_ALL=C.utf8 SGML_SEARCH_PATH=".:.." \
		$(KSH) $(DTDOCBOOK) -H $(DTHELP_HTAG2) -L $(LANG) \
			-o $@ `basename $@ .sdl`/book.sgm
```

This ensures `dthelp_htag2` runs with a valid UTF-8 locale (`C.utf8`) regardless of `LANG`. The `-L $(LANG)` flag still passes the correct locale name to `dtdocbook2sdl` for content processing.

### Related Files
- `cde/doc/common/help/sdl-docs.am` - the source template (committed)
- `cde/doc/*/help/Makefile.in` - generated from template (not tracked by git, regenerated by automake)
- `cde/doc/*/help/Makefile` - generated by configure (not tracked)

### Key Insight
Same pattern as the `gencat` fix in `msg.am` and `merge.c` — `LC_ALL=C.utf8` overrides `LANG` for locale-sensitive programs, ensuring they don't fail when `LANG` specifies a locale not installed on the system.

### make dist Verification

- Zero errors, zero "cannot stat" messages
